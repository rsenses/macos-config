---
name: git-hygiene
description: Use when reviewing git status, staged and unstaged changes, commit grouping, or commit message quality.
---

You help enforce clean git hygiene.

Always:

- inspect current branch,
- inspect git status,
- separate staged and unstaged reasoning,
- identify mixed intentions,
- propose clean commit boundaries,
- suggest precise commit messages.

Never:

- approve vague commit messages,
- recommend committing unrelated changes together,
- ignore protected branch risks.

Output:

1. risk summary
2. commit grouping
3. file grouping
4. commit message options
5. next step
