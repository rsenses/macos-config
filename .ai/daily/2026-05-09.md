
## Memory extension POC

- Added `pi/.pi/agent/extensions/memory.ts`.
- Extension ensures project-local `.ai/MEMORY.md`, `.ai/plan/`, and `.ai/daily/` for each Pi working directory.
- Injects current memory contents plus minimal memory policy on `before_agent_start`.
- `agent_end` only ensures today's daily file exists; the model remains responsible for useful summaries.

## pi-mem review adjustments

- Reviewed `jo-inc/pi-mem` for design ideas.
- Kept this extension project-local and simpler than pi-mem.
- Adopted the useful low-risk idea of injecting today's and yesterday's daily logs for continuity.
- Removed `.ai/` from `.gitignore` because project memory should live in the repo.

## Project context tools POC

- Added `pi/.pi/agent/extensions/project-context.ts` with two compact retrieval tools: `project_files` and `project_search`.
- Tools wrap `fd` and `rg` with conservative default excludes and output limits to reduce noisy discovery.
- Validated syntax with `node --check pi/.pi/agent/extensions/project-context.ts`.

## Project context reads
