description: Review branch and repository state before starting a task
---

Use the `ops` skill.

1. **Git State**: Inspect current branch, git status, and recent commits.
2. **Analysis**: Detect if the branch is appropriate and if the working tree is clean.
3. **Recommendation**: Suggest a branch name if needed.
4. **Plan**: Provide a very brief starting strategy if requirements are clear.

*Note: If the branch is protected or the tree is dirty, state it first.*
