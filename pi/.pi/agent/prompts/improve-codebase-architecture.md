Use the `ops` and `dev` skills.

Analyze the codebase architecture ($ARGUMENTS) to identify friction points and suggest practical improvements.

**Rules**:
1. Do not modify code or create files.
2. Base recommendations on evidence from the codebase (use `scout`).
3. Follow project patterns. Prefer framework-native solutions.
4. Distinguish between real friction and harmless style differences.

**Output**:
- **Summary**: Brief diagnosis.
- **Evidence**: Files/patterns inspected.
- **Friction Points**: What makes development harder and why.
- **Recommendations**: Small, incremental, and reversible improvements.
- **Next Step**: One practical action.
